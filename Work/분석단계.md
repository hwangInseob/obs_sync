이 모듈은 다행히 복잡한 상태 관리 라이브러리(Redux 등)를 사용하지 않고, 자체적인 [AppData.js](cci:7://file:///c:/Users/User/Desktop/HomeIOT/source/home_iot_dev/web/src/wrm/STH/data/AppData.js:0:0-0:0) 클래스와 기본적인 React 웹 생태계를 바탕으로 구성되어 있습니다. 다음 순서대로 학습하고 코드를 따라가 보시는 것을 추천합니다.

### 1단계: React 필수 개념 (이 모듈을 이해하기 위한 최소한의 지식)

React의 모든 것을 다 알 필요는 없습니다. 현재 모듈 구조를 볼 때 다음 3가지만 먼저 이해하시면 됩니다.

*   **컴포넌트 (Component)와 Props**:
    *   `src/wrm/STH/screens/` 아래의 파일들(`Main.js` 등)이 각각 하나의 화면(컴포넌트)입니다.
    *   컴포넌트들은 레고 블록처럼 조립되며, 부모 컴포넌트가 자식 컴포넌트에게 데이터를 전달할 때 **Props**라는 것을 사용합니다. (예: `<ShortcutButton title="바로가기" />`에서 `title`이 Prop입니다.)
*   **상태 (State)와 `useState` Hook**:
    *   화면상에서 변하는 데이터(예: 서버에서 받아온 방문자 목록, 로딩 상태 등)를 관리합니다. 데이터가 바뀌면 화면이 자동으로 다시 그려집니다(렌더링).
*   **생명주기 (Lifecycle)와 `useEffect` / `useLayoutEffect` Hook**:
    *   컴포넌트가 화면에 나타날 때(Mount), 혹은 특정 데이터가 바뀔 때 어떤 동작을 할지 정의합니다.
    *   `routes.js`의 `RootScreen` 컴포넌트에서 화면 구성을 시작할 때 `useLayoutEffect`를 사용해 `AppData.reset()`을 호출하는 것이 대표적인 예시입니다.

### 2단계: 모듈 전용 패턴 파악하기 (STH 코드 읽는 법)

STH 모듈만의 고유한 패턴 세 가지를 숙지하시면 코드가 훨씬 잘 읽힙니다.

1.  **화면 이동은 어떻게 하는가? (`routes.js`)**
    *   URL에 따라 어떤 화면 컴포넌트(`screens/...`)가 열릴지 정의되어 있습니다.
    *   코드 내부에서 화면을 이동할 때는 `State.go('STH_CarHistory')` 와 같은 방식으로 이동하는 패턴을 씁니다. (`Main.js` 참고)
2.  **데이터는 어디서 가져와서 어떻게 저장하는가? (`AppData.js`)**
    *   일반적인 React 앱들은 상태 관리 도구(Redux, Zustand 등)를 쓰지만, STH 모듈은 **`AppData.js`라는 자체 클래스 내의 Map 객체(`_homes`, `_sites` 등)에 데이터를 캐싱(저장)합니다.**
    *   화면이 켜지면 `AppData.init()`을 호출해 내 집 정보와 단지 정보를 싹 다 가져와서 캐싱해 둡니다.
3.  **서버 API 호출은 어떻게 하는가? (`useAppService` & `ServiceApis.js`)**
    *   데이터가 필요할 땐 직접 서버 주소를 적지 않습니다.
    *   `ServiceApis.js` 에 백엔드 서버와 통신하는 함수(`getCarHistory`, `getVisitors` 등)들이 정의되어 있으며, 이들은 자동으로 필요한 암호화 헤더 등을 붙여줍니다.
    *   화면(UI) 컴포넌트에서는 `useAppService` 훅을 사용해 데이터를 편하게 가져옵니다.
